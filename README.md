이 프로젝트는 HTML로 먼저 정리한 테트리스 개발 계획서를 바탕으로, 실제 플레이가 가능한 Python/Pygame 테트리스 게임으로 확장한 결과물이다.

처음 계획서에서는 테트리스의 기본 구조를 중심으로 정리했다.
위에서 떨어지는 블록을 조작해 빈틈 없이 쌓고, 가득 찬 줄을 지워 점수를 올리는 방식, 7종 블록의 회전 상태, 격자판 구조, 충돌 판정, 줄 삭제, 점수 처리, 게임 루프, 다음 블록 표시 같은 핵심 요소를 먼저 글과 화면 스케치로 구성했다.
이후 실제 코드에서는 그 내용을 바탕으로 플레이 가능한 형태의 테트리스를 구현했고, 여기에 메뉴와 난이도 같은 확장 기능도 추가했다.

1. 게임의 기본 개념 반영
   - 이 게임은 HTML 계획서에서 정리한 테트리스의 기본 개념을 그대로 중심축으로 삼아 구현되었다.
   - 블록이 위에서 내려오고,
   - 플레이어가 이를 좌우로 이동하거나 회전시키고,
   - 줄이 가득 차면 삭제되며,
   - 점수와 레벨이 함께 올라가는 구조로 만들어졌다.
   - 코드에서는 이러한 핵심 구조가 블록 데이터, 이동/회전 처리, 줄 삭제, 점수 반영 로직으로 나누어 실제 동작하도록 구성되어 있다.

2. 색상과 도형 기반 표현
   - 계획서에서 정리했던 것처럼, 이 프로젝트는 이미지 파일 없이 Pygame의 사각형 그리기 기능을 사용해 모든 블록과 보드를 표현한다.
   - 각 블록은 고유 색상을 가지며, 보드 위에 놓인 블록, 현재 떨어지는 블록, 다음 블록 미리보기까지 모두 동일한 방식으로 직접 그려진다.
   - 즉, 별도의 리소스 파일 없이도 하나의 Python 파일만으로 실행 가능한 구조를 갖는다.

3. 화면 구성
   - 화면은 크게 왼쪽의 플레이 영역과 오른쪽의 정보 패널로 나뉜다.
   - 왼쪽에는 실제 테트리스 보드와 현재 블록이 표시되고,
   - 오른쪽에는 점수, 라인 수, 레벨, 난이도, 다음 블록 3개, 조작 안내가 배치된다.
   - 이 구조는 HTML 계획서에 있던 화면 스케치를 실제 UI 형태로 옮긴 것이다.

4. 7종 블록과 회전 상태
   - 계획서에서 정리했던 I, O, T, S, Z, J, L 7종 블록이 모두 구현되어 있다.
   - 각 블록은 회전 상태를 가지며, 플레이 중 회전 키 입력에 따라 형태가 바뀐다.
   - 계획서에서는 BLOCK_DATA 형태로 설명되었지만, 실제 코드에서는 문자열 기반 블록 데이터 구조를 사용해 같은 개념을 더 다루기 쉽게 표현했다.

5. 게임 상태 관리 구조
   - 계획서에서는 FIELD, BLOCK, NEXT_BLOCK 같은 전역 변수 중심으로 설명되었지만,
     실제 구현에서는 TetrisGame 클래스 안에 게임 상태를 모아 관리하는 구조로 바뀌었다.
   - 현재 블록, 다음 블록 큐, 보드 상태, 점수, 라인, 레벨, 게임오버 여부 같은 값들이 하나의 게임 객체 안에서 함께 관리된다.
   - 이를 통해 코드 구조를 더 명확하게 정리하고, 기능 확장도 쉽게 할 수 있게 만들었다.

6. 랜덤 블록 생성
   - 게임이 시작되면 새로운 블록이 랜덤하게 등장한다.
   - 실제 구현에서는 7-bag 방식이 사용되어, 특정 블록이 과하게 반복되거나 한동안 전혀 나오지 않는 문제를 줄였다.
   - 이 부분은 계획서의 “랜덤 블록 등장” 개념을 실제 플레이 품질까지 고려해 확장한 구현이다.

7. 이동, 회전, 낙하, 충돌 처리
   - 블록은 좌우 이동, 빠른 낙하, 하드 드롭, 회전이 가능하다.
   - 이동과 회전은 항상 충돌 판정을 먼저 거친 뒤 적용되며,
   - 벽, 바닥, 이미 쌓인 블록과 겹칠 경우에는 해당 동작이 막힌다.
   - 이 부분은 계획서에서 설명했던 충돌 판정과 조작 개념이 실제 플레이 로직으로 구현된 핵심 구간이다.

8. 줄 삭제 기능
   - 보드의 한 줄이 완전히 채워지면 그 줄은 삭제되고, 위에 있던 블록들이 아래로 내려온다.
   - 계획서에서 설명했던 줄 삭제 로직이 실제 코드에서도 동일한 목적을 가지도록 구현되었다.
   - 플레이 중 가장 기본이 되는 테트리스의 핵심 규칙 중 하나다.

9. 블록 고정과 다음 블록 등장
   - 블록이 더 이상 아래로 내려갈 수 없으면 현재 위치에 고정된다.
   - 이후 줄 삭제 여부를 검사하고,
   - 점수와 라인 수를 갱신한 뒤,
   - 다음 블록이 새롭게 등장한다.
   - 이 과정은 계획서의 “착지 → 고정 → 줄 삭제 → 다음 블록 생성” 흐름을 실제 게임 안에서 한 번에 처리하는 핵심 로직이다.

10. 점수 시스템
    - 계획서에서는 줄 수에 따라 점수를 단순 계산하는 방식으로 설명했지만,
      실제 코드에서는 좀 더 일반적인 테트리스 점수 방식에 가깝게 조정했다.
    - 한 번에 여러 줄을 지울수록 더 큰 점수를 얻을 수 있고,
    - 레벨이 높아질수록 획득 점수도 함께 증가한다.
    - 이를 통해 단순히 오래 버티는 것뿐 아니라, 효율적으로 줄을 지우는 플레이도 중요하게 만들었다.

11. 레벨 증가와 속도 변화
    - 계획서에서 정리한 난이도 상승 구조는 실제 코드에서 레벨 시스템으로 구현되었다.
    - 일정 수의 줄을 지우면 레벨이 오르고,
    - 레벨이 오를수록 블록의 자동 낙하 속도도 빨라진다.
    - 즉, 게임이 진행될수록 플레이어는 더 빠른 판단과 조작을 요구받게 된다.
    - 이는 일반적인 테트리스의 진행 방식과 유사한 구조다.

12. 다음 블록 미리보기
    - 계획서에는 다음 블록 미리보기 기능이 포함되어 있었고,
      실제 코드에서는 이를 더 확장해 다음 블록을 3개까지 볼 수 있도록 만들었다.
    - 이를 통해 플레이어가 미리 블록 흐름을 파악하고 전략적으로 배치할 수 있다.

13. 게임오버 처리
    - 새 블록이 등장해야 하는 위치가 이미 막혀 있으면 게임은 종료된다.
    - 이 부분은 계획서의 “맨 위까지 쌓이면 게임 오버” 개념을 실제 구현에 맞게 좀 더 정확한 방식으로 처리한 것이다.
    - 게임오버가 되면 화면에 안내가 표시되고, 설정에 따라 다시 시작할 수 있다.

14. 게임 루프 구조
    - 이 프로젝트는 Pygame의 기본 구조에 맞춰
      “이벤트 처리 → 상태 업데이트 → 화면 그리기”
      순서의 게임 루프로 동작한다.
    - 키 입력과 마우스 입력을 처리하고,
    - 블록 상태와 보드 상태를 갱신한 뒤,
    - 매 프레임마다 현재 화면을 다시 그리는 구조다.
    - 이 흐름은 HTML 계획서에서 설명했던 게임 루프 개념을 그대로 실제 코드로 옮긴 것이다.

15. 계획서 기반 핵심 기능 구현 여부
    - HTML 계획서에서 정리했던 테트리스의 핵심 기능들은 대부분 실제 코드에 반영되었다.
    - 창 생성
    - 7종 블록
    - 자동 낙하
    - 좌우 이동
    - 빠른 낙하
    - 회전
    - 착지 후 고정
    - 줄 삭제
    - 게임오버 처리
    - 점수 / 레벨 / 다음 블록 표시
    같은 요소들이 모두 실제 게임 플레이에 연결되어 있다.
    - 즉, 이 프로젝트는 단순한 아이디어 정리에 그치지 않고, 계획서를 실제 코드로 확장한 형태라고 볼 수 있다.

16. 계획서 이후 추가된 확장 기능
    - 이 프로젝트에는 HTML 계획서에는 없었지만, 이후 추가로 확장된 기능들도 포함되어 있다.
    - 대표적으로는 다음과 같다.
      - 메뉴 시스템
      - 쉬움 / 보통 / 어려움 난이도 선택
      - 시작 레벨 설정
      - 도움말 페이지
      - 스크롤 가능한 안내 창
      - 키보드와 마우스를 모두 지원하는 메뉴 조작
      - 다음 블록 3개 미리보기
      - 착지 예측 위치(고스트 블록) 표시 및 난이도에 따른 표시 여부 변경
      - 게임오버 시 재시작 / 설정에 따른 플레이 중 리셋 허용
    - 이 기능들은 기본 테트리스 구조 위에 사용자 편의성과 플레이 다양성을 더하기 위해 추가된 요소들이다.

정리하면, 이 프로젝트는 먼저 작성한 HTML 계획서의 핵심 내용을 실제 플레이 가능한 테트리스 게임으로 구현한 결과물이다.
기본적인 블록, 보드, 충돌, 줄 삭제, 점수, 레벨, 게임 루프 같은 핵심 요소는 계획서의 흐름을 따라 반영되었고,
그 위에 메뉴, 난이도, 도움말, 마우스 조작, 고스트 블록 같은 기능을 추가해 더 완성도 있는 형태로 확장했다.

즉,
이 프로젝트는 단순히 테트리스를 한 번 만들어본 코드가 아니라,
“계획서를 먼저 만들고, 그 구조를 실제 게임으로 구현하고, 이후 기능을 확장해 나가는 방식”으로 진행한 개발 과정이 함께 담긴 테트리스 프로젝트라고 소개할 수 있다.
